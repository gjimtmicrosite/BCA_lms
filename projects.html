🔎 Project 1: Algorithm Logic Comparison Google Benchmark Weeks 1–6

Objective: Compare different C-based algorithmic approaches for the same problem and justify which approach is better in terms of clarity, correctness, performance and robustness.

Theme: “Think like Google when comparing algorithms for the same problem.”

AI Tools: ChatGPT / Gemini for alternative logic ideas, time-space explanation, comparison tables and code review support.

🏢 Benchmark Narrative

  • Google must choose efficient and scalable algorithms for search, autocomplete, recommendation, ranking and spam detection.
  • Students mirror this practice by comparing multiple C solutions to the same problem.
  • The focus is not only writing code, but also explaining why one logic is better than another.

📋 4-Phase Framework

🟠 Manual work first · 🤖 AI augmentation after manual attempt · Final decision must be student’s own reasoning.

🟠 Phase 1 — Problem Selection & Manual Understanding
  • Select one C programming problem suitable for multiple logic approaches.
  • Suggested problem: Find the top 3 largest numbers in a list.
  • Write the input, output, constraints and expected behaviour manually.
  • Deliverable: Problem definition sheet + sample inputs and outputs.
🟠 Phase 2 — Manual Logic Design
  • Design at least 3 approaches manually before using AI.
  • Example Approach A: Sort the array and pick the last 3 values.
  • Example Approach B: Single pass maintaining top 3 values.
  • Example Approach C: Partial selection logic.
  • Deliverable: Flowchart / pseudocode for all approaches.
🟠 Phase 3 — C Coding, Testing & Comparison
  • Write C code for all selected approaches.
  • Test with multiple inputs including duplicate values, negative numbers, small arrays and sorted/unsorted arrays.
  • Create comparison table: readability, number of loops, expected performance, edge-case handling and memory use.
  • Deliverable: C source files + test case sheet + comparison table.
🤖 Phase 4 — AI Lab & Google-Style Decision
  • Use ChatGPT/Gemini to ask for alternative logic and edge cases.
  • Use AI to explain time and space complexity in simple language.
  • Ask AI to review your comparison table, but do not copy the final answer blindly.
  • Final student decision: Which approach would you “ship” if you were in Google’s engineering team?
  • Deliverable: Final report + AI reflection note + viva.

🧠 Sample AI Prompts

  • “Generate three different C logic approaches to find the top 3 largest numbers in an array. Explain each approach simply.”
  • “Compare sorting-based logic and single-pass logic for large input size. Which is more efficient and why?”
  • “Review my C code and suggest edge cases I should test.”

🪟 Project 2: Utility Program Development Microsoft Benchmark Weeks 7–12

Objective: Build a small menu-driven C utility that is correct, user-friendly, robust and properly documented.

Theme: “Build a small utility like a junior developer at Microsoft working on Windows/Office tools.”

AI Tools: GitHub Copilot for coding assistance, ChatGPT for code review and edge cases, Notion AI/Grammarly for documentation.

🏢 Benchmark Narrative

  • Microsoft builds many small utilities such as calculators, file explorers, configuration wizards and productivity helpers.
  • Students create a simple C utility that resembles a real-world user tool.
  • The focus is clean menu design, input validation, helpful messages and clear documentation.

📋 4-Phase Framework

🟠 Manual planning first · 🤖 AI support for improvement · Student must understand and explain every code block.

🟠 Phase 1 — Utility Selection & Requirement Planning
  • Select a simple utility program idea.
  • Recommended example: Student Gradebook Utility.
  • Define features: Add record, View all records, Search by roll number, Calculate grade, Exit.
  • Deliverable: Requirement specification sheet + menu sketch.
🟠 Phase 2 — Manual Program Design
  • Create flowchart and pseudocode for the complete menu-driven program.
  • Decide data structure: arrays, structures, loops and functions.
  • Plan validation rules for marks, roll number, empty records and wrong menu choices.
  • Deliverable: Flowchart + function list + validation checklist.
🟠 Phase 3 — C Development & Testing
  • Write the C program with functions and comments.
  • Add clear prompts and helpful error messages.
  • Test with valid and invalid input cases.
  • Prepare screenshots of each menu option and output.
  • Deliverable: C source code + screenshots + testing sheet.
🤖 Phase 4 — AI Review, Documentation & Microsoft-Style Polish
  • Use Copilot as a pair programmer for boilerplate, loops and menu structures.
  • Use ChatGPT as a code reviewer to catch edge cases and improve prompts.
  • Use Notion AI/Grammarly to prepare user guide and project documentation.
  • Final program must be understandable, tested and well-documented.
  • Deliverable: Final C utility + user guide + AI reflection note + viva.

🧠 Sample AI Prompts

  • “Review this C menu-driven program and identify possible input validation issues.”
  • “Suggest better user-friendly messages for a student gradebook utility.”
  • “Create a simple user guide for this C utility including purpose, features and steps to use.”
