📊 Project Evaluation Rubrics

P1 — Algorithm Logic Comparison Google Benchmark

CriteriaWeightExcellentGoodNeeds Work
Problem Understanding25%Clear input/output, constraints and edge cases.Mostly clear problem with basic I/O.Vague or missing constraints.
Algorithm Comparison30%Compares 3 approaches with time/space and tests.Compares 2–3 approaches with limited explanation.Only one approach or weak comparison.
C Implementation & Testing25%Clean C code, multiple test cases, edge cases handled.Code works for normal cases.Incomplete or poorly tested code.
AI Reflection & Justification20%Strong AI reflection and final “ship” decision.Basic reflection and choice.AI copied without judgement.

P2 — Utility Program Development Microsoft Benchmark

CriteriaWeightExcellentGoodNeeds Work
Utility Design25%Clear purpose, user-friendly menu and complete flow.Menu and purpose clear with limited features.Unclear purpose or menu flow.
C Programming Quality30%Modular code, validation, error messages, clean functions.Working code with basic validation.Code has errors or weak structure.
Documentation20%Complete user guide, screenshots, testing table and comments.Basic documentation and comments.Missing or poor documentation.
AI-Assisted Improvement25%AI used for review/debugging with human corrections.AI used for basic improvement.AI copied blindly or not used meaningfully.
